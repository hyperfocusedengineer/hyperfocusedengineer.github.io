The Idea

The UI is the output

Most AI tools give you a chatbot. Gumball gives you a workspace. Tell it "track Austin housing for me" and you don't get a text response — you get a signal card showing buy/wait, a metrics dashboard with median price and mortgage rates, a 20-week price trend chart, and a ranked list of matching properties. All generated by Claude, all rendered as native mobile components.

The key insight: different tasks need different interfaces. A tweet composer needs draft variants, character counts, and tone pickers. A YouTube script needs expandable sections, research sources, and a hook editor. Gumball's block system makes this possible — Claude picks the right blocks for each agent's job.

Core Concept

Task-Aware UI — 11 Block Types

Each agent's workspace is composed of blocks. Claude selects 3–6 blocks per agent based on what the agent does. Blocks are self-contained UI components with their own data.

Design

7 Color Moods

Each agent can have its own color scheme. Claude auto-suggests a mood based on the agent's domain, or users can pick one manually. The mood tints the entire workspace — signal cards, chart bars, hook backgrounds, metric values, and selection chips all respond to it.

Features

What's built

Conversational Creation
Multi-turn dialogue with Claude to refine your agent. Smart questions with suggested defaults. Build when you're ready.
Versioned Editing
Edit agents in natural language. Each edit creates an immutable version snapshot. Full history, rollback.
Model Selection
Haiku 4.5 (fast), Sonnet 4 (balanced), Opus 4 (powerful). Global default with per-agent override.
Zod Validation
Every Claude API response validated with Zod schemas. Malformed responses caught before they hit the UI.
Offline Detection
Banner shows when you're offline. Prevents wasted API calls and confused users.
Crash Reporting
Sentry integration for production visibility. Disabled in dev, 20% trace sampling.
